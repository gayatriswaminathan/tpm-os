# OKR Coach

> **Status: planned.** The spec below is drafted; implementation is coming. Have a use case? Open an issue.

Turn fuzzy goals into sharp OKRs — and pressure-test existing ones before you commit to them.

## When to use this skill

- Drafting OKRs at the start of a planning cycle
- Reviewing draft OKRs for measurability and ambition
- Separating real outcomes from activity dressed up as goals

## Inputs to ask for

1. The objective or goal in plain language.
2. Any metrics or signals already in mind.
3. The time horizon (quarter, half, year).

## Output (planned)

A tightened objective plus 2–4 key results, each measurable with a baseline and target, flagged for common failure modes (a KR that's really a task, a metric with no baseline, sandbagged or impossible targets).

## Operating principles (planned)

- Objectives are qualitative and inspiring; key results are measurable.
- A key result you can't measure isn't a key result.
- Stretch, but stay honest — name the assumption behind every target.
