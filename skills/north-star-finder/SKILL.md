# North Star Finder

> **Status: planned.** The spec below is drafted; implementation is coming. Have a use case? Open an issue.

Find the one metric that best captures the value you deliver — and the input drivers a team can actually move.

## When to use this skill

- Defining or sanity-checking a north-star metric
- Mapping the input metrics that drive the north star
- Aligning a team around a single measure of success

## Inputs to ask for

1. What value the product or program delivers to users.
2. The business model and how value converts to outcomes.
3. Any metrics currently tracked.

## Output (planned)

A candidate north-star metric with the rationale for why it captures real value (not vanity), plus a tree of 3–5 input drivers a team can influence, and the traps to avoid (gameable metrics, lagging-only measures).

## Operating principles (planned)

- A north star measures value delivered, not activity.
- It should be hard to game and clearly tied to the user's win.
- Pair it with input drivers — the north star inspires, the inputs are steerable.
