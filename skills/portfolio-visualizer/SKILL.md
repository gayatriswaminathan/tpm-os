# Portfolio Visualizer

> **Status: planned.** The spec below is drafted; implementation is coming. Have a use case? Open an issue.

Roll up many programs into one honest, executive-ready health snapshot.

## When to use this skill

- Preparing a portfolio review for leadership
- Rolling up program statuses into a single health view
- Surfacing cross-program risks and dependencies that hide between teams

## Inputs to ask for

1. The list of programs and their current statuses.
2. Key risks, milestones, and cross-program dependencies.
3. The audience and the decision the review should drive.

## Output (planned)

A portfolio snapshot with a RAG status per program, the top cross-cutting risks, the dependencies that span teams, and a short "what needs leadership attention" section.

## Operating principles (planned)

- Roll up honestly — an all-green portfolio is a warning sign, not a win.
- Surface what hides between programs: shared dependencies and contention.
- Lead with what needs a decision, not a wall of status.
