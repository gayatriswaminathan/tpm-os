// quickstart

Clone. Customize. Run.

1

Pick a skill

Open its SKILL.md to see what it does and what inputs it expects.

2

Point it at your context

Drop it into Claude, Copilot, or an MCP server and feed it your own data.

3

Ship the artifact

A clean, audience-ready output in seconds — judgment still yours.

status-update
risk-tracker
prd-generator
> use skill: status-update

audience: exec
program: Payments Migration
notes: |
  finished 12 of 18 migrations
  staging down 3 days, blocking tests

# → RAG-rated exec update,
# every risk with an owner + date.
> use skill: risk-tracker

program: Payments Migration
notes: |
  vendor api may slip again
  staging down, blocking testing
  pen test needed before go-live

# → scored RAID log: probability,
# impact, mitigation, owner, date.
> use skill: prd-generator

idea: export dashboard data to CSV
who: data-team analysts
constraints: web only, ship this quarter

# → PRD with goals, non-goals,
# metrics, and acceptance criteria.
